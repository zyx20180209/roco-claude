# 对战回放记录（网页版）

边看高分局视频/直播，边按回合记录双方动作和状态变化。数据存在浏览器本地，可导出 JSON 离线分析。

**在线使用：** https://zyx20180209.github.io/roco-claude/apps/replay_recorder/

---

## 设计目标

- **零服务器依赖**：纯前端 + localStorage，GitHub Pages 直接托管
- **以精灵为单位记录**：每只精灵在每回合的状态变化独立成条
- **结构化数据**：导出的 JSON 可直接喂给后续分析脚本（`analysis/state_builder.py`）

---

## 快速上手

### 1. 队伍设置

打开页面后填入：

- **回放 ID**（如 `r001`，用作导出文件名，留空会用时间戳）
- **我方 6 只精灵**（填入后首发下拉框自动更新）
- **敌方 6 只精灵**
- **我方/敌方首发精灵**（从下拉框选择）

点击 **开始记录**。

> 中途关闭浏览器没关系，下次打开点 **恢复上次** 即可继续（数据存在 localStorage）。
> 也可以点 **导入 JSON** 从已有回放文件继续记录。


---

### 2. 回合记录

每个回合点 **+ 添加变化**，依次为本回合发生变化的精灵填表单。

#### 表单分区

**精灵**
- 方：我方 / 敌方
- 精灵：选择具体精灵
- 换入 / 换出：本回合是否上下场
- 首领化：本回合是否首领化（每场战斗每只精灵最多一次，不占用回合）

**技能与血量**
- 使用技能：技能名（可空，如换人/聚能）
- 血量（我方）：当前血量 / 血量上限（数字）
- 血量（敌方）：百分比（0-100）
- 能量Δ：能量变化（如 `-5` 表示消耗5能，`+5` 表示聚能）

**受到的伤害**
- 直接伤害：技能攻击造成的伤害数字
- DoT 伤害：本回合末 DoT 造成的伤害数字
- 被击杀：本回合是否阵亡

**DoT 层数变化**
- 毒Δ / 灼烧Δ / 冻结Δ：层数变化（如 `+2` 表示新增2层）

**Buff / Debuff 变化**（临时，换出消失）
- 物攻Δ / 魔攻Δ / 物防Δ / 魔防Δ / 速度Δ / 连击Δ / 威力Δ：层数变化

**印记变化**
- 新增 / 移除：印记名（逗号分隔，如 `风起印记,光合印记`）

**永久效果变化**（萌化/特性提升等，换出不消失）
- 物攻Δ% / 魔攻Δ% / 物防Δ% / 魔防Δ%：百分比加成
- 速度Δ / 威力Δ / 连击Δ：固定值加成
- 萌化Δ：萌化层数

#### 操作流程

1. 点 **+ 添加变化**，填一只精灵的本回合状态
2. 点 **确认**，回到回合面板
3. 重复 1-2，把本回合所有精灵的变化都加上
4. 每条变化下面有 **备注** 输入框，可补充非标信息（如"对方应对成功"）
5. 点 **编辑** 重新填表，点 **×** 删除
6. 全部加完后点 **提交本回合**，状态面板自动更新

---

### 3. 状态面板

提交回合后，左右两侧会显示双方 6 只精灵的当前状态：

- **血量条**：绿色 > 50%，橙色 > 25%，红色 ≤ 25%
- **HP 文本**：我方显示 `当前/上限`，敌方显示百分比
- **能量**：⚡ 标签
- **DoT**：红色标签（毒/灼/冻）
- **Buff**：绿色标签（增益）/ 紫色标签（减益）
- **永久效果**：青色标签（如 `物攻+30%`、`萌+1`）
- **印记**：黄色标签
- **首领**：金色边框 + 金色"首领"标签
- **在场**：蓝色边框
- **战死**：半透明

---

### 4. 操作栏

- **提交本回合**：把当前变化写入 turns，更新状态面板，回合末自动结算灼烧减半
- **回到上回合**：撤销最后提交，把那回合的变化放回编辑区继续修改
- **导出 JSON**：下载完整记录文件（用于后续分析）
- **重置**：清空所有数据，回到队伍设置页

---

### 5. 回合历史

提交后，回合历史区显示每回合的详细变化（精灵、技能、HP、能量、伤害、buff/dot/印记等），每回合右侧有 **返回此回合** 按钮，点击后进入编辑模式：

- 状态面板回到该回合**之前**的状态
- 之后的回合不删除
- 标题变金色"编辑第N回合"，按钮变"保存修改"
- 修改完成后点"保存修改"，替换该回合并重算所有后续状态
- 点"取消编辑"放弃修改

---

### 6. 战斗结束

在变化表单的"受到的伤害"区勾选 **战败** 或 **投降**，提交后历史区顶部显示战斗结果横幅。


## 换出时的状态清空规则

精灵换出后自动处理：

| 状态 | 是否清空 |
|------|---------|
| 临时 buff/debuff | ✅ 清空（力量增效/加速等） |
| 毒 / 灼烧 | ✅ 清空 |
| 冻结 | ❌ 保留 |
| 永久效果 | ❌ 保留（萌化、特性带来的属性提升） |
| 印记 | ❌ 保留（印记是场地层面，不随精灵走） |

## 数据格式

导出的 JSON 结构：

```json
{
  "id": "r001",
  "created_at": "2026-04-29T10:00:00",
  "my_team": ["精灵1", "精灵2", "精灵3", "精灵4", "精灵5", "精灵6"],
  "enemy_team": ["精灵1", "精灵2", "精灵3", "精灵4", "精灵5", "精灵6"],
  "turns": [
    {
      "turn": 1,
      "changes": [
        {
          "side": "my",
          "pokemon": "寂灭骨龙",
          "skill": "闪击",
          "hp_cur": 320,
          "hp_max": 380,
          "hp_pct": 84,
          "energy_delta": -5,
          "buffs": {"attack": 1},
          "note": "应对成功"
        },
        {
          "side": "enemy",
          "pokemon": "黑猫巫师",
          "hp_pct": 60,
          "damage_received": {"direct": 472, "dot": null},
          "dots": {"burn": 2}
        }
      ]
    }
  ]
}
```

### 字段说明

每条 `change` 的字段（只填有变化的）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `side` | `"my"` / `"enemy"` | 所属方 |
| `pokemon` | string | 精灵名 |
| `entered` | bool | 本回合换入 |
| `exited` | bool | 本回合换出 |
| `fainted` | bool | 本回合战死 |
| `boss` | bool | 本回合首领化 |
| `skill` | string | 使用的技能 |
| `hp_cur` / `hp_max` | int | 我方：当前/上限血量 |
| `hp_pct` | float | 敌方：血量百分比；我方自动从 cur/max 推算 |
| `energy_delta` | int | 能量变化（±） |
| `damage_received` | `{direct, dot}` | 受到的伤害（数值） |
| `buffs` | object | 临时 buff 层数变化 |
| `dots` | object | DoT 层数变化（poison/burn/freeze） |
| `marks_added` / `marks_removed` | string[] | 印记变更 |
| `passive_effects` | object | 永久效果变化（attack_pct, speed_flat, moe 等） |
| `note` | string | 备注 |
