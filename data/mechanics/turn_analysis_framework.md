# 回合分析框架

本文档定义了对每回合双方决策进行系统性分析的方法。配合 [`decision_value.md`](./decision_value.md) 使用。

---

## 一、分析的输入

每个回合需要以下信息：

| 类别 | 内容 |
|------|------|
| **回合状态** | 双方在场精灵、HP、能量、buff/debuff/dot/印记/永久效果 |
| **双方行动** | 使用的技能 / 换人 / 聚能 / 首领化 / 愿力冲击 |
| **行动结果** | 造成的伤害（直接+DoT）、能量变化、状态变化 |
| **静态数据** | 双方精灵的种族值、属性、技能学习池、**特性（含能耗减少/永久效果等被动）** |
| **环境数据** | 当前印记、天气、其他全局效果 |

---

## 二、分析的步骤

每回合按以下步骤分析，**禁止跳步**：

### 步骤 1：列出可选动作

对每方精灵，枚举本回合所有合法动作：

- 4个技能（能量足够的）
- **注意特性/被动对能耗的影响**（如疾风连袭在特定条件下减少能耗，可能让"能量不足"的技能变为合法）
- 聚能（始终可用）
- 换人到场下任意存活精灵
- 首领化（每场最多一次）
- 愿力冲击（与首领化互斥）

### 步骤 2：分析双方意图

不要直接评价"操作好坏"，先**推断双方为什么这么选**：

| 决策类型 | 常见意图 |
|---------|---------|
| 攻击 | 直接斩杀 / 累计伤害 / 测试对方应对 |
| 状态 | 铺设印记 / 强化自身 / 限制对方 / 挂DoT逼换 |
| 防御 | 应对预判攻击 / 等待DoT斩杀 / 触发应对效果 |
| 换人 | 规避必死 / 建立优势对位 / 主动用残血换换人机会 / **预判对手换人目标** |
| 聚能 | 能量不足 / 预判对方使用状态/防御 |
| 首领化/愿力 | 关键回合的爆发 / 应对状态触发增益 |

**换人预判分析（新增）：**
当一方选择攻击技能时，需额外分析是否存在"预判对手换人"的意图：
- 对手队伍中哪只精灵最可能换入？（血量充裕、克制当前对位、有迅捷技能）
- 当前技能对该精灵是否有克制？
- 若有克制，该攻击选择可能是针对换入精灵的预判，而非针对当前在场精灵

**核心原则：玩家的每个动作都有意图，不要默认是"失误"**。如果某个动作看起来不合理，先问"如果我是玩家，我为什么这么选？"

### 步骤 3：核对应对/速度

判断本回合的出手顺序：

1. 是否存在应对关系？（防御应对攻击、攻击应对状态等）
2. 应对成功方先手且获得增益
3. 无应对时比较速度（含 buff/passive_effects/印记加成）

### 步骤 4：核对能量管理

每个动作的能量代价不能忽略：

- **能耗效率**：相同伤害下，谁消耗的能量更少？
- **能量储备**：双方剩余能量能支撑几回合？
- **能量差**：哪一方更可能因为能量不足被迫聚能？

### 步骤 5：核对属性克制

伤害计算必须考虑属性克制：

- **强制查表**：每次涉及伤害的回合，**必须查 `data/raw/type_chart.json`**，不能凭印象
- 单属性：直接乘法（如 水→机械 = 2x）
- 双属性：克制乘法，但**双克制 = 3x（不是4x），双抗 = 1/3（不是1/4）**
  - 例：恶→龙(1x) × 恶→幽(2x) → 恶→龙/幽 = 2x
  - 例：水→机械(2x) × 水→水(1x) → 水→机械/水 = 2x
  - 例：水→机械(2x) × 水→火(2x) → 水→机械/火 = 3x（双克封顶3x）
- 部分免疫为 0

### 步骤 6：评估短期收益（本回合）

| 收益项 | 衡量方式 |
|--------|---------|
| 血量差 | 双方造成的伤害（含DoT） |
| 能量差 | 双方消耗的能量（防御应对成功 = 对方能量浪费） |
| 状态差 | 新增的 buff/debuff/印记/dot 净值 |
| 精灵数 | 是否有精灵战死 |

### 步骤 7：评估长期收益（多回合）

| 收益项 | 衡量方式 |
|--------|---------|
| 永久效果 | 萌化、首领化加成、应对成功的永久收益（如水刃免费化）|
| **永久效果量化** | 永久免费技能 = 每回合节省的能量 × 预期剩余回合数；例：水刃永久免费=每回合+4能等效，10回合=40能等效收益 |
| 印记累积 | 场地印记的回合数收益（光合每回合1能、风起每回合+20%）|
| 阵容针对性 | 当前对位 + 场下精灵的克制关系 |
| 资源换资源 | 主动用残血精灵换换人机会，避免后续被动 |

### 步骤 8：识别关键判断点

每回合可能有 1-2 个**关键判断**，决定胜负走向：

- **应对触发**：水刃应对状态成功 → 永久免费输出
- **斩杀线突破**：本回合首次能斩杀对方
- **优势对位建立**：换人后克制+速度双优势
- **关键能量节点**：对方刚耗光能量，必须聚能

### 步骤 9：避免事后诸葛

- **不要用后续回合的信息评判当前决策**——玩家做决策时不知道未来
- **不要假设玩家知道对方的精确数据**——很多时候是预判和读心
- **如果某个动作"看起来失误"，优先怀疑分析框架不完整**，再考虑是失误

## 三、输出格式

每回合的分析按以下结构输出：

```markdown
### 第N回合：[一句话概括]

**局面：**
- 我方：[在场精灵] HP/能量/buff/dot
- 敌方：[在场精灵] HP/能量/buff/dot
- 场地：[印记]

**可选动作：**
- 我方：[列举主要选项]
- 敌方：[列举主要选项]

**实际行动：**
- 我方：[实际选择]
- 敌方：[实际选择]

**双方意图分析：**
- 我方意图：[推断]
- 敌方意图：[推断]

**应对/速度：**
[出手顺序判定]

**能量核对：**
[双方能量变化]

**属性克制：**
[本回合涉及的克制关系]

**收益评估：**
- 短期：[血量/能量/状态/精灵数]
- 长期：[永久效果/印记/阵容]

**关键判断（如有）：**
[识别本回合的胜负决定点]

**评分：** [我方优 / 敌方优 / 均衡 / 难判] - [一句话理由]
```

## 四、常见误判模式

避免以下分析错误：

### 1. 忽略意图推断
> ❌ "黑羽夫人立刻换出，不合理"
>
> ✅ "黑羽夫人换出可能是预判敌方继续用状态技能，让圣羽翼王也获得永久免费水刃"

### 2. 凭印象判断属性克制
> ❌ "水对机械无克制"
>
> ✅ "查表：水克制机械，2x 伤害"

### 3. 忽略能量管理
> ❌ "齿轮切开 vs 气泡，双方互相重创"
>
> ✅ "齿轮切开 5 能，气泡 3 能，敌方能量效率更高"

### 4. 把残血换人当失误
> ❌ "尖嘴狐仙换入即战死，是失误"
>
> ✅ "尖嘴狐仙残血+中毒，主动换入承受一击，换取寂灭骨龙满血上场——合理的残血换人战术"

### 5. 用后续信息倒推决策
> ❌ "他应该早点换出寂灭骨龙"（玩家当时不知道会被斩杀）
>
> ✅ "寂灭骨龙留场是因为对岚鸟有抵抗，但低估了岚鸟撕咬的连击数"

## 五、特殊状态速查表

分析前必须确认以下状态的正确机制：

| 状态 | 机制 | 换人行为 | 常见误判 |
|------|------|---------|---------|
| 毒 | 每回合末扣最大HP×3%×层数 | 随精灵下场消失 | 误以为换人保留 |
| 灼烧 | 每回合末扣最大HP×2%×层数，**回合末层数减半（向下取整）** | 随精灵下场消失 | 误以为不衰减 |
| 冻结 | **降低血量上限**（每层-5%斩杀线），当前HP低于斩杀线立即斩杀 | **换人不消失** | 误以为是DoT伤害 |
| 萌化 | **精灵退化到上一阶段，种族值降低，但特性和技能不变**；破罐破摔等技能在自身有减益时威力提升 | 随精灵下场消失 | 误以为只是攻击力下降，忽略种族值全面降低的影响 |

## 六、待补充的分析维度

随着实战数据积累，以下维度可能需要补充到框架：

- [ ] 主播视角的"读心"：基于对手历史习惯预判
- [ ] 时间压力（对战剩余时间）对决策的影响
- [ ] 心态因素（连胜/连败对决策风险偏好的影响）
- [ ] 队伍体系的整体定位（速攻/控制/续航/输出）
