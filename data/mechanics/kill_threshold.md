# 秒杀阈值分析

从伤害公式推出的**秒杀阈值模型**，用于快速判断"一击秒杀某只精灵"需要什么样的攻击值和技能威力。

---

## 核心公式

### 伤害公式

```
伤害 = CEIL(0.9 × 连击数 × 有效威力 × 攻击 / 防御)
```

### 秒杀条件

```
伤害 ≥ HP
  ↓
0.9 × 攻击 × 有效威力 / 防御 ≥ HP
  ↓
攻击 × 有效威力 ≥ HP × 防御 / 0.9
```

### 秒杀阈值 T

```
T = HP × 防御 / 0.9
```

**只要攻击能力值 × 有效威力 ≥ T，就能一击秒杀。**

连击数视为"威力乘子"的一部分，直接算进有效威力即可：`有效威力 = 基础威力 × 连击数 × ...`

---

## 有效威力的组成

有效威力 = 基础威力 × 所有加成的乘积。常见加成：

| 加成项 | 系数 | 说明 |
|--------|------|------|
| 本系加成 (STAB) | ×1.25 | 使用与自身属性相同的技能 |
| 属性克制 | ×2 / ×3 | 单克/双克（防御方双抗为 ×1/2 或 ×1/3） |
| 应对成功 | ×1.5 | 愿力冲击应对状态成功；其他"应对成功"技能的额外收益 |
| 连击数 | ×2 / ×3 | 描述中写明"N 连击"的技能 |
| 特性加成 | ×1.1 ~ ×1.5 | 如目空（非光系+25%）、夺目、顺风(+50%)、变形活画（每层+10%）等 |
| 印记加成 | ×1.1 ~ ×1.3 | 攻击印记(+10%/层)、蓄势印记(+30%)、风起印记(+20% 先手) |
| 天气加成 | ×1.5 | 雨天对水系技能 |
| 基础威力修正 | +20 等 | 折射搭配普通系时威力+20（加在基础威力上） |

**示例：** 彩虹独角兽(首领化)的追打（普通系，75 威力）+ 折射附带的"普通+20威力"+ 夺目特性（非光系+25%）+ 翼系传来的连击+1：
- 有效威力 = (75+20) × 1.25 × 2 = 237.5

## 使用流程

### 1. 确定防御方

需要 3 个参数：**HP、对应防御值、是否加 HP 性格/个体**。

防御方通常按最常见的耐久配置算：**HP/物防/魔防 + 加 HP 性格**。

### 2. 计算阈值 T = HP × 防御 / 0.9

### 3. 按场景反推攻击值

给出几种典型"有效威力"场景，反推每种场景需要的攻击能力值：

| 场景 | 有效威力 |
|------|---------|
| 纯愿力冲击应对成功 | 80 × 1.5 = 120 |
| 愿力冲击 + 本系 | 120 × 1.25 = 150 |
| 愿力冲击 + 本系 + 单克 | 120 × 1.25 × 2 = 300 |
| 愿力冲击 + 本系 + 双克 | 120 × 1.25 × 3 = 450 |
| 150 威力本系技能 + 双克 | 150 × 1.25 × 3 = 562.5 |
| 追打 3 连（应对） + 夺目 | 75 × 3 × 1.25 × 1.25 ≈ 352 |

然后 `需要攻击 = CEIL(T / 有效威力)`，再反查需要的种族值。

### 4. 判断可行性

- 全游戏最高物攻种族值：烈火战神 175 → 加攻满个体 = **333**
- 全游戏最高魔攻种族值：彩虹独角兽 192 → 加攻满个体 = **355**

如果反推的所需攻击值 > 这两个上限，则**当前环境无法一击秒杀**，只能依赖多次输出、DoT 或冻结斩杀等手段。

## 实用脚本

位置：`scripts/tools/kill_threshold.py`

```python
from scripts.tools.kill_threshold import (
    analyze_defender,   # 计算阈值 T
    scenario_table,     # 列出场景 → 所需攻击 → 所需种族
    power_grid,         # 给定攻击值反推所需威力
)

hp, df, T = analyze_defender(
    "翠顶夫人 魔防方向",
    hp_base=125, hp_iv=10, hp_nature="加",
    def_base=97, def_iv=10, def_nature="无",
    def_label="魔防",
)

scenario_table(T, [
    ("愿力冲击 有本系 2x", 120 * 1.25 * 2.0),
    ("150威力本系 3x",     150 * 1.25 * 3.0),
])
```

## 备注

1. **性格按项生效**：加攻性格只加物攻，不加魔攻/速度/防御。反推种族值时注意攻击类型与性格是否匹配。
2. **个体分配限制**：每只精灵最多只有 3 项能有个体值加成。默认假设攻击方"攻击/速度/HP 点满"，防御方"HP/物防/魔防 点满"。
3. **取整误差**：公式中的 ROUND/CEIL 累积误差约 ±1~2 血，阈值判断结果偏差在此范围内属正常。
4. **阈值不是线性**：HP 和 防御都是**乘积关系**。把敌方血量压到 70%（如用一次消耗技能）就能让阈值降到 70%，显著提高可秒杀性。
